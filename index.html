Simulation model catalog vocabulary

MoCAT is a vocabulary for describing simulation models. This document lists its requirements, defines the schema and provides examples.

A simulation model is a tool to make a digital representation of a real-world phenomenon in nature, within a population, in a transport network, etc.. We can use simulation models to better understand observations of the past or to study the impact of changes in the future.

A common vocabulary for describing simulation models on the Web enhances its discoverability and automation potential.

Requirements

The following table summarizes the use cases for a simulation model catalog.
Summary Description Priority
Discover simuation models Users must be able to find and compare relevant simulation models based on domain, purpose, legal constraints, ownership, licensing, quality assessments, validation reports, and compliance with internal or regulatory standards,reducing duplication of effort and improving transparency across organizations. High
Test interoperability Metadata enables automated reasoning about which models can use which datasets (and vice versa) by describing input/output data structures, semantics, spatial/temporal resolution, units, and licensing constraints, allowing systems to automatically match or flag incompatibilities. Medium
Reproduce a simulation run Metadata captures version history, parameter sets, data provenance, and software dependencies so that simulation results can be reproduced, audited, or certified, even when original creators are no longer involved. [[PROV-O]] High

Simulation Model

A simulation model is a tool to make a digital representation of a real-world phenomenon.

This digital representation has a theoretical and mathematical basis. The simulation model is usually implemented in source code and software.

Data Specification

A data specification describes how data is structured.

Each input and output of a simulation model can have its own data specification. The data is not necessarily structured as RDF. In the case of RDF data, [[SHACL]] can be used.
